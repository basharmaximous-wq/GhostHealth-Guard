⚙️ Multi-Layered Security

01
Static Privacy Scanning

Custom Semgrep rules catch hardcoded sensitive strings and unmasked logging of PHI before they ever reach review.

02
AI-Powered Code Review

Specialized LLM (Gemini 1.5 Flash / GPT-4o) hunts for "Privacy Smells" — unencrypted transit, improper Debug trait on sensitive data.

03
Architectural PHI Tracking

Vector Database (RAG) understands your codebase's data models. Knows if a struct is #[Sensitive] and flags unsafe handling across the entire repo.

04
Confidential Computing

Optional Intel SGX enclave support for processing audit logs in a secure, hardware-isolated environment. Even Root can't touch it.

05
Immutable Audit Chain

Every scan result notarized on a blockchain and linked via ZK-proofs — your compliance history is tamper-proof for auditors, forever.

🛠️ Installation

⚡ Quick Start — No Rust Required

Download the binary for your OS from the Latest Release, extract, and move to path:

ghosthealth-guard --version

🦀 Build from Source

git clone https://github.com/basharmaximous-wq/GhostHealth-Guard.git
  cd GhostHealth-Guard
  cargo install --path .

🔧 Environment Variables (.env)

DATABASE_URL=postgres://user:pass@localhost:5432/ghostdb
GEMINI_API_KEY=your_gemini_api_key_here
GITHUB_APP_ID=your_app_id
PRIVATE_KEY_PATH=path/to/key.pem

🗄️ Initialize Database

sqlx migrate run

💻 Run a Local Scan

ghosthealth-guard scan ./src --api-key $GEMINI_API_KEY

🔗 GitHub Actions Integration (.github/workflows/ghosthealth_check.yml)

name: Compliance Audit
on: [pull_request]

jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run GhostHealth Guard
        run: ghosthealth-guard scan-pr --pr-number ${{ github.event.number }}
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}

❓ Troubleshooting & FAQ

Missing API Key?

Ensure GEMINI_API_KEY or OPENAI_API_KEY is exported in your environment before running any scan commands.

Timeout on Large Repos?

Use the --timeout flag to increase processing time for deep architectural analysis across large codebases.

Getting a False Positive?

Suppress specific warnings using the #[allow(compliance_risk)] or #[ignore_compliance] attributes on your functions.

Database Connection Refused?

GhostHealth Guard requires a live PostgreSQL instance. Ensure your DATABASE_URL is correct and migrations have been run with sqlx migrate run.
